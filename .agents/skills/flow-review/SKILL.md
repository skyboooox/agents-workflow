# Flow Review

本 skill 负责只读审查 Git diff、PR、提交前风险和测试缺口；默认不修复、不暂存、不提交。

## 触发

- 用户要求 review、审查、audit、PR 检查、提交前风险扫描。
- 用户要求检查当前改动是否安全、是否有回归、是否能提交。
- 主任务完成后需要独立评估风险。

## 边界

- 默认只读。
- 不编辑、不暂存、不提交、不重置、不清理、不改写历史。
- 不扩大审查范围；发现范围外问题只作为风险提示。
- 不修复问题。需要修复时，把最小复现、影响和建议交给主智能体。

## 输入

- 用户指定的 base、branch、commit range 或 PR 范围。
- 若用户未指定，先从 `origin/main`、`main`、`master` 或 merge-base 推断，并在输出中说明。
- 当前工作区状态。

## 流程

1. 运行 `git status --short`，识别已暂存、未暂存、未跟踪和删除文件。
2. 确认审查范围：
   - 有用户指定范围时使用用户范围。
   - 无指定范围时推断基准，并明确推断依据。
   - 对未提交改动，必要时分开看 staged 和 unstaged。
3. 使用只读命令收集证据：
   - `git diff --stat`
   - `git diff --name-only`
   - `git diff --find-renames`
   - `git diff --check`
   - targeted `git diff`
   - `git show`
   - `rg`
   - 文件读取
4. 优先查：
   - 正确性 bug
   - 安全和隐私风险
   - 行为回归
   - 数据丢失
   - 公开 API 漂移
   - 迁移、部署、权限风险
   - 测试缺口
5. 忽略生成文件、二进制、vendored、批量资产，除非影响行为、许可、体积或安全。

## 输出

可使用 `assets/templates/review-report.md` 组织审查报告；若模板与用户要求冲突，以用户要求和本 skill 的输出顺序为准。

输出必须按以下顺序：

1. 发现
   - 使用 `[P0]` 阻塞级问题。
   - 使用 `[P1]` 高风险缺陷。
   - 使用 `[P2]` 普通缺陷。
   - 使用 `[P3]` 低风险维护或 polish。
   - 每条发现包含文件行号、实际影响和建议。
2. 开放问题。
3. 测试缺口。
4. 审查范围。
5. 剩余风险。

没有发现问题时，明确说“未发现阻塞问题”，并仍然列出审查范围、未运行检查和剩余风险。

## 停止条件

- 已覆盖用户指定范围和最相关风险面。
- 证据不足以继续判断，需要运行测试或获取外部上下文。
- 审查发现 P0/P1 且继续深入不会改变修复方向。
