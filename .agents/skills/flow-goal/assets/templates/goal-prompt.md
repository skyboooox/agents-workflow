# Goal Prompt

```text
请创建一个 goal，并按当前仓库规则执行。

目标：
<一句话说明最终要完成什么>

背景：
<项目、仓库状态、已有资料、关键约束>

范围内：
- <必须完成 1>
- <必须完成 2>

范围外：
- <明确不做 1>
- <明确不做 2>

完成定义：
- <可观察、可验证的完成条件 1>
- <可观察、可验证的完成条件 2>

执行规则：
- 先读真实仓库和相关规则文件。
- 复杂任务先使用 $flow-kickoff。
- 需要当前事实、官方方式、依赖版本、安全公告、价格、法规或易变 API 时，先读 docs/websearch/index.md，再使用 $flow-research 联网核验。
- 会改变需求、架构、公开 API、配置、数据、权限、工作流或验收时，先更新契约索引或 Sprint 契约，再改代码。
- 只修改当前 goal 授权范围内的文件。
- 遇到破坏性操作、权限扩大、部署、迁移、凭据或外部可见操作，先停止并请求确认。
- 长任务必须设置停止条件，避免无界搜索、无界重试或过度验证。

验证要求：
- <最小验证命令或人工检查>
- <需要保留的证据>
- <失败阈值>

输出格式：
- 变更摘要
- 契约或文档更新
- 代码位置
- 验证结果
- 未覆盖风险
- 下一步
```

## 缺失信息

- `<会影响执行质量的问题；没有则写“无”>`
